What is an Observable?
  • An observable basically can be thought of as a data source.
  • In our angular project, an observable basically just is an object, we import from third party package, RXJS.
  • The observable here is implemented in a way that it follows the observable pattern, (so we have ab observable and we have an observer.)
  • In between we have a time line.
  • In this time line we have multiple events emitted by the observable or data package
  • Observable is for the consumer, it can be transformed and subscribed:
    • observable.map(x => ...).filter(x => ...).subscribe(x => ...)

    What is observer?

  • Observer is the interface which is used to feed an observable source
    • observer.next(newItem)
  • In this we can handle data package in three ways
    • Handle Data
    • Handle Error
    • Handle Completion
  • We use the patterns to handle asynchronous tasks. (as that http request/response)
